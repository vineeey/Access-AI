# --- Wake word + voice commands (Phase 10) ---
# TORCH SAFETY: openWakeWord is a pure-Python CPU detector that runs its
# pretrained models through the EXISTING onnxruntime - it does NOT depend on
# torch, so it cannot move torch off 2.4.1. It was installed with
#   pip install --no-deps openwakeword==0.6.0

#   System lib for the live mic:  sudo apt install -y libportaudio2
openwakeword==0.6.0

# --- Natural human-like TTS (Phase 11) ---
# TORCH SAFETY: this phase replaces the robotic espeak/pyttsx3 voice WITHOUT
# touching the pinned torch 2.4.1. That is exactly why we use the ONNX build:
#   * kokoro-onnx  -> runs Kokoro through the EXISTING onnxruntime (a Phase-2
#                     dep). It does NOT depend on torch. The STANDARD `kokoro`
#                     pip package DOES depend on torch and can silently move it
#                     (it broke YOLO in Phase 7) - DO NOT install that one.
#   * edge-tts     -> Microsoft Neural voices over pure async HTTP. No torch.
#   * soundfile    -> decodes edge-tts MP3 output to samples AND writes the temp
#                     WAV we hand to the OS player (bundles libsndfile 1.1.0).
# Playback runs in a SHORT-LIVED SUBPROCESS (aplay/paplay/pw-play/ffplay), NOT an
# in-process PortAudio stream: sounddevice's ALSA backend corrupts the process
# heap ("double free ... in pa_linux_alsa.c") when a playback stream is torn down
# repeatedly in a long-lived server. A subprocess player isolates that crash. No
# extra pip dep - one of those players ships with almost every desktop Linux.
# Verify torch is STILL 2.4.1 and YOLO still detects after installing these.
#
# !! CRITICAL - kokoro-onnx 0.4.9 DECLARES numpy>=2.0.2 AND onnxruntime>=1.20.1,
#    which would UPGRADE the pinned numpy 1.26.4 / onnxruntime 1.18.1 and can
#    silently break YOLO/insightface (the Phase-7 failure mode). Those pins are
#    over-tight: kokoro-onnx is pure-Python ONNX glue and runs fine on our stack.
#    So install it WITH --no-deps and add its safe deps explicitly:
#      pip install edge-tts==6.1.19 soundfile==0.12.1 colorlog>=6.9.0 \
#                  espeakng-loader>=0.2.4 phonemizer-fork>=3.3.2 \
#                  numpy==1.26.4 onnxruntime==1.18.1
#      pip install --no-deps kokoro-onnx==0.4.9
#    Then re-check: torch==2.4.1 and YOLO bus.jpg still detects (NOT zero).
#
# Kokoro model files (offline voice) are NOT on PyPI - they download ONCE (~336MB
# total) into  models/kokoro/  from the kokoro-onnx GitHub releases:
#   kokoro-v1.0.onnx  (~310MB):
#     https://github.com/thewh1teagle/kokoro-onnx/releases/download/model-files-v1.0/kokoro-v1.0.onnx
#   voices-v1.0.bin   (~26MB):
#     https://github.com/thewh1teagle/kokoro-onnx/releases/download/model-files-v1.0/voices-v1.0.bin
# If that download can't happen, TTS degrades to edge-tts (online) or pyttsx3
# (offline espeak) automatically - the system never goes silent.
#   kokoro-onnx==0.4.9   (install with --no-deps, see the CRITICAL note above)
edge-tts==6.1.19
soundfile==0.12.1
colorlog>=6.9.0
espeakng-loader>=0.2.4
phonemizer-fork>=3.3.2
